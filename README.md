# Лабораторная работа: Генерация и тестирование ПСП

Самодостаточный проект на **C++17**. Три кастомных генератора псевдослучайных последовательностей, статистический анализ, критерий Хи-квадрат, пять NIST-тестов и бенчмарк.

## Структура папки

```
psp_lab/

├── generate_psp_plots.py   # Построение графиков из CSV

    ├── nist_results.csv
    ├── histogram.csv
    ├── psp_report.html   # HTML-отчёт с графиками
    └── plots/            # PNG-графики
```

## Сборка и запуск

```powershell
cd psp_lab
g++ -std=c++17 -O2 -o psp_lab.exe main.cpp
```

## Генераторы — подробно

Все генераторы реализуют единый интерфейс `IGenerator`:

| Метод | Назначение |
|-------|------------|
| `next(min, max)` | Целое число в диапазоне [min, max] — для статистики и бенчмарка |
| `nextRaw()` | Сырое 32-битное значение — для побитовых NIST-тестов |
| `seed(s)` | Сброс состояния (воспроизводимость экспериментов) |

Нормализация в диапазон [0, 5000]: `result = min + (raw % (max - min + 1))`.

---

### 1. LCG — линейный конгруэнтный генератор

**Идея:** классический рекуррентный метод, используемый ещё в ранних системах (glibc `rand()`, старые игры).

**Формула:**

\[
X_{n+1} = (a \cdot X_n + c) \bmod m
\]

| Параметр | Значение | Смысл |
|----------|----------|-------|
| a | 1 103 515 245 | Множитель |
| c | 12 345 | Приращение (чтобы не застревать в нуле) |
| m | 2³¹ = 2 147 483 648 | Модуль |

**Как работает:** каждый вызов `nextRaw()` обновляет `state_` по формуле и возвращает его. Период не превышает m.

**Плюсы:** очень быстрый (одно умножение + сложение + остаток), простая реализация.

**Минусы:** низшие биты имеют короткий период; последовательные значения коррелированы; битовая структура слабая — Monobit и Runs часто отклоняются.

---

### 2. Von Neumann + Weyl — метод середины квадрата

**Идея:** предложен фон Нейманом в 1949 году. Берётся число, возводится в квадрат, из середины результата берутся цифры как следующее значение.

**Базовый алгоритм (без модификации):**

\[
X_{n+1} = \text{middle\_digits}(X_n^2)
\]

**Проблема:** быстро **вырождается в ноль** (например, 1² = 1, 0² = 0) или зацикливается на малых числах.

**Модификация Вейля** (добавлена в нашей реализации):

\[
X_{n+1} = \text{middle}(X_n^2 + w_n), \quad w_{n+1} = w_n + \delta
\]

где \(\delta = \texttt{0x9E3779B97F4A7C15}\) — константа Вейля (связана с золотым сечением). На каждом шаге к квадрату прибавляется «шум», который никогда не повторяется, — это разрушает циклы нуля.

**Извлечение средних битов:** из 64-битного `(state² + weyl)` берутся биты [16..47] — 32 средних бита.

**Плюсы:** нет умножения на константу a; непредсказуемая нелинейность от возведения в квадрат.

**Минусы:** медленнее LCG/Xorshift; качество сильно зависит от seed; распределение в [0, 5000] может быть приемлемым, но биты всё ещё не идеальны.

---

### 3. Xorshift32 — сдвиговый регистр

**Идея:** George Marsaglia, 2003. Состояние — одно 32-битное число. Обновляется тремя XOR и битовыми сдвигами — без умножения.

**Формула:**

```
state ^= state << 13;
state ^= state >> 17;
state ^= state << 5;
```

**Как работает:** сдвиги «перемешивают» биты; при правильных константах (13, 17, 5) период = 2³² − 1.

**Плюсы:** быстрый, компактный (4 байта состояния), хорошее качество для некриптографических задач.

**Минусы:** не криптостойкий; при плохом seed возможны слабые начальные значения (поэтому seed = 0 заменяется на `0xDEADBEEF`).

---

### 4. std::mt19937 (эталон для бенчмарка)

Вихрь Мерсенна из `<random>`. Не «наш» генератор — используется только для сравнения скорости со стандартом C++.

## Как проверялось качество

Проверка шла в **два уровня**: равномерность чисел в диапазоне [0, 5000] и случайность битов.

### Уровень 1 — описательная статистика (20 выборок × 1000 чисел)

Для каждой выборки считались:

| Метрика | Теория для U[0, 5000] | Что смотрели |
|---------|----------------------|--------------|
| Среднее | 2500 | Близко ли к центру диапазона |
| Дисперсия | ≈ 2 083 333 | Разброс значений |
| СКО | ≈ 1443 | √дисперсии |
| CV | ≈ 57.7% | (СКО/среднее)·100% |

**Результаты:** все три генератора давали среднее 2400–2600, CV ≈ 55–60% — согласуется с равномерным распределением.

### Уровень 2 — критерий Хи-квадрат (равномерность)

- Диапазон [0, 5000] разбит на **10 карманов** по 500 чисел
- Ожидаемая частота в каждом: 1000 / 10 = **100**
- Статистика: \(\chi^2 = \sum \frac{(O_i - E)^2}{E}\)
- Степени свободы: 9, α = 0.05, критическое значение: **16.919**

| Генератор | Принято выборок | Вердикт |
|-----------|-----------------|---------|
| LCG | 19/20 (95%) | Хорошо |
| Von Neumann+Weyl | 20/20 (100%) | Отлично |
| Xorshift32 | 18/20 (90%) | Хорошо |

Отклонённые выборки: LCG #8 (χ²=19.24), Xorshift #2 (χ²=18.5), Xorshift #14 (χ²=22.54) — единичные аномалии при 95% доверии нормальны.

### Уровень 3 — NIST-тесты (битовая случайность)

10 000 сырых 32-битных чисел → 320 000 бит. Пять тестов из NIST SP 800-22:

| Тест | Что проверяет | LCG | Von Neumann | Xorshift |
|------|---------------|-----|-------------|----------|
| Monobit | Баланс 0 и 1 | ✗ | ✓ | ✓ |
| Frequency in Block | Доля единиц в блоках 128 бит | ✓ | ✓ | ✓ |
| Runs | Длины серий одинаковых битов | ✗ | ✓ | ✓ |
| Longest Run of Ones | Макс. серия единиц в блоке | ✗ | ✗ | ✗ |
| Serial | Частоты пар битов | ✗ | ✗ | ✓ |

**Итого NIST:** LCG 1/5, Von Neumann 3/5, Xorshift 4/5.

### Почему результаты различаются?

- **Хи-квадрат** проверяет: «числа в [0, 5000] распределены равномерно?»
- **NIST** проверяет: «отдельные биты ведут себя как подбрасывание монеты?»

Генератор может давать равномерные числа в диапазоне, но при этом иметь **слабую битовую структуру** (типично для LCG). Это не противоречие — разные аспекты качества.

### Уровень 4 — бенчмарк производительности

Замер с `std::chrono`, 3 прогрева, `volatile sink` против оптимизаций компилятора.

На 1 000 000 элементов (мс):

| Генератор | Время |
|-----------|-------|
| LCG | ~3.3 |
| Von Neumann | ~6.0 |
| Xorshift | ~4.0 |
| std::mt19937 | ~5.4 |

**Вывод:** LCG — самый быстрый, но слабее по битам; Xorshift — лучший баланс скорости и качества; Von Neumann — медленный, нестабильный по NIST.
